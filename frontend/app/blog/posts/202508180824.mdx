## 📌 [moeru-ai/airi](https://github.com/moeru-ai/airi)
<Callout>
    Description: 💖🧸 Self hosted, you owned Grok Companion, a container of souls of waifu, cyber livings to bring them into our worlds, wishing to achieve Neuro-sama's altitude. Capable of realtime voice chat, Minecraft, Factorio playing. Web / macOS / Windows supported.\
    🌐 Vue｜⭐️ 8,476 | 5373 stars this week
</Callout>
    #### 簡介

Project AIRI 是一個旨在重新創造 Neuro-sama 的開源專案，目標是打造一個可在多平臺執行的 AI 虛擬角色，讓使用者擁有自己的數位生命伴侶。它結合了大型語言模型 (LLM) 的能力，並支援遊戲、聊天等多種互動方式，並強調使用 Web 技術以及跨平臺相容性。不同於其他專案，AIRI 從一開始就著重於 Web 技術 (WebGPU, WebAudio, Web Workers, WebAssembly, WebSocket 等)，讓它可以在現代瀏覽器和行動裝置上執行，同時也支援桌面版本，利用 native NVIDIA CUDA 和 Apple Metal 提升效能。目前專案仍在早期開發階段，積極尋求開發者加入。


#### 主要功能

Project AIRI 的主要功能包含：

*   **腦 (Brain):**  整合大型語言模型 (LLM)，支援多個 LLM API 提供商，例如 OpenAI、Google Gemini 等。  能進行聊天互動，並可與遊戲互動 (Minecraft, Factorio)。
*   **記憶 (Memory):** 使用內建瀏覽器資料庫 (DuckDB WASM, pglite) 和 Memory Alaya (開發中) 儲存資訊。
*   **耳朵 (Ears):** 支援從瀏覽器和 Discord 接收音訊輸入，並進行客戶端語音辨識和說話偵測。
*   **嘴巴 (Mouth):** 使用 ElevenLabs 語音合成進行語音輸出。
*   **身體 (Body):** 支援 VRM 和 Live2D 模型，並具備自動眨眼、注視和閒置眼動等動畫功能。
*   多平臺支援：瀏覽器 (PWA)、桌面應用程式。
*   豐富的外掛系統 (WIP)。
*   多種遊戲整合：Minecraft, Factorio (部分功能)。


#### 如何使用

目前 Project AIRI 仍處於開發階段，使用方式主要透過開發者模式參與：

*   **安裝套件:** 使用 `pnpm i` 安裝專案所需的套件。
*   **啟動開發伺服器:** 使用 `pnpm dev` 啟動瀏覽器版本 (Stage Web) 的開發伺服器；使用 `pnpm dev:tamagotchi` 啟動桌面版本 (Stage Tamagotchi) 的開發伺服器；使用 `pnpm dev:docs` 啟動檔案網站的開發伺服器。
*   參考 `CONTRIBUTING.md` 檔案，瞭解專案的開發流程和環境設定。
*   貢獻程式碼：參與專案開發，擴充套件 AIRI 的功能。
*   目前無公開釋出的使用者版本。

  專案包含許多子專案，提供多種工具和函式庫，例如：語音轉文字、文字轉語音的通用端點代理伺服器 (unspeech)、HuggingFace Spaces 部署工具 (hfup) 等。
## 📌 [simstudioai/sim](https://github.com/simstudioai/sim)
<Callout>
    Description: Sim is an open-source AI agent workflow builder. Sim's interface is a lightweight, intuitive way to rapidly build and deploy LLMs that connect with your favorite tools.\
    🌐 TypeScript｜⭐️ 11,712 | 4321 stars this week
</Callout>
    #### 簡介

Sim 是一個能讓使用者在幾分鐘內建置和部署 AI 代理工作流程的平臺。它提供雲端託管和自託管兩種部署方式，支援使用本地模型 (例如透過 Ollama)，並整合了向量嵌入功能，例如知識庫和語義搜尋。Sim 使用 PostgreSQL 資料庫 (建議使用 pgvector 擴充套件)，並需要 Bun runtime 環境。


#### 主要功能

* 建置和部署 AI 代理工作流程。
* 雲端託管 (sim.ai) 和自託管選項。
* 支援使用 Ollama 執行本地 AI 模型，無需外部 API。
* 使用向量嵌入進行知識庫和語義搜尋，需要 PostgreSQL 資料庫 (建議使用 pgvector 擴充套件)。
* 提供多種自託管部署方式：NPM 套件、Docker Compose 和開發容器。
* 整合 Copilot 服務 (需 API 金鑰)。
* 提供開發模式，包含主應用程式和即時 Socket 伺服器。


#### 如何使用

* **雲端託管:** 直接訪問 sim.ai。
* **自託管 (NPM):** 使用 `npx simstudio` 命令。需安裝 Docker。
* **自託管 (Docker Compose):**  
    * 複製 GitHub 倉庫: `git clone https://github.com/simstudioai/sim.git`
    * 使用 `docker compose -f docker-compose.prod.yml up -d`  啟動。
* **自託管 (Ollama):** 使用 `docker compose -f docker-compose.ollama.yml`  啟動，支援 GPU 和 CPU 版本。
* **自託管 (開發容器):** 使用 VS Code 的 Remote - Containers 擴充套件。執行 `bun run dev:full`。
* **自託管 (手動):**  需要 Bun runtime 和 PostgreSQL 12+ (含 pgvector 擴充套件)。需設定 `.env` 檔案，並執行 `bunx drizzle-kit migrate` 和 `bun run dev:full` (或分別啟動 Next.js 應用程式和即時 Socket 伺服器)。  PostgreSQL 和 pgvector 的安裝可選擇 Docker 或手動安裝。
## 📌 [coleam00/Archon](https://github.com/coleam00/Archon)
<Callout>
    Description: Beta release of Archon OS - the knowledge and task management backbone for AI coding assistants.\
    🌐 Python｜⭐️ 10,467 | 4256 stars this week
</Callout>
    #### 簡介

Archon 是一個 beta 版本的 AI 程式碼助理命令中心，提供簡潔的介面來管理專案的知識、上下文和任務。它作為一個模型上下文協定 (MCP) 伺服器，讓多個 AI 程式碼助理 (例如 Claude Code、Kiro、Cursor 等) 能同時存取相同的知識庫、上下文和任務，提升 AI 程式碼生成的效率。Archon 不僅能管理現有程式碼庫，也能應用於新專案，其知識和任務管理功能能改善任何 AI 驅動的程式碼編寫輸出。


#### 主要功能

* **知識管理:** 支援智慧型網頁爬取、檔案處理 (PDF、Word 檔案、Markdown 等)、程式碼範例萃取、向量搜尋及來源管理。
* **AI 整合:** 提供模型上下文協定 (MCP) 讓多種 AI 程式碼助理連線，支援 OpenAI、Ollama 和 Google Gemini 模型，並提供多種 RAG 策略。
* **專案與任務管理:** 提供階層式專案組織、AI 輔助的專案與任務建立、檔案版本控制和協同編輯、以及進度追蹤。
* **即時協作:** 使用 WebSocket 更新提供即時進度追蹤，支援多使用者協同作業，並具備背景處理和健康監控功能。


#### 如何使用

* **設定:**  需要 Docker Desktop、Node.js 18+ (混合開發模式)、Supabase 帳號和 OpenAI API 金鑰 (或其他支援的 LLM)。複製 repo、設定 .env 檔案 (包含 Supabase 憑證)、設定資料庫 (執行 `migration/complete_setup.sql`)。
* **啟動服務:** 使用 `docker compose --profile full up --build -d` 或 `make dev-docker` 啟動 Docker 容器 (伺服器、MCP 伺服器、代理程式和 UI)。
* **設定 API 金鑰:**  瀏覽器開啟 http://localhost:3737，前往設定頁面設定 LLM/嵌入式提供者的 API 金鑰。
* **測試:**  上傳檔案、爬取網站，建立專案並新增任務，以及將其與 AI 程式碼助理整合。
* **資料庫重置:**  如有需要，可執行 `migration/RESET_DB.sql` 重設資料庫，再執行 `migration/complete_setup.sql` 重新建立資料表，並重新啟動服務和設定。
* **開發模式:** 使用 `make install` 安裝依賴項，並使用 `make dev` 或 `make dev-docker` 啟動混合開發或全 Docker 開發模式。
## 📌 [emcie-co/parlant](https://github.com/emcie-co/parlant)
<Callout>
    Description: LLM agents built for control. Designed for real-world use. Deployed in minutes.\
    🌐 Python｜⭐️ 7,444 | 3495 stars this week
</Callout>
    #### 簡介

Parlant 是一個用於開發 AI 代理程式的框架，解決了傳統 LLM 代理程式在遵循指令、處理邊緣情況和保持一致性方面存在的問題。與傳統方法依靠複雜且難以管理的系統提示不同，Parlant 允許開發者以自然語言定義規則和行為準則，確保 AI 代理程式能準確地遵循預期的流程。它提供了建立符合業務需求的客戶端代理程式的結構，並能有效地避免 LLM 產生幻覺或忽略指令。Parlant 的目標是讓 AI 代理程式的開發更可靠、更可預測，並更容易擴充套件。


#### 主要功能

* **Journeys (旅程):** 定義清晰的客戶旅程，並指定代理程式在每個步驟中的回應方式。
* **Behavioral Guidelines (行為準則):** 以自然語言輕鬆制定代理程式行為；Parlant 會根據上下文匹配相關元素。
* **Tool Use (工具使用):** 將外部 API、資料擷取器或後端服務附加到特定的互動事件。
* **Domain Adaptation (領域適應):** 教導代理程式特定領域的術語，並制定個性化的回應。
* **Canned Responses (預設回應):** 使用回應範本來消除幻覺並保證風格的一致性。
* **Explainability (可解釋性):** 瞭解每個準則何時以及為何被匹配和遵循。
* **Conversational Journeys (對話旅程):** 引導客戶逐步完成目標。
* **Dynamic Guideline Matching (動態準則匹配):** 基於上下文的規則應用。
* **Reliable Tool Integration (可靠的工具整合):** API、資料庫、外部服務。
* **Conversation Analytics (對話分析):** 深入瞭解代理程式行為。


#### 如何使用

Parlant 提供簡潔的 Python SDK，方便開發者快速建立和部署 AI 代理程式。  以下是一個簡單的範例：

```python
import parlant.sdk as p

@p.tool
async def get_weather(context: p.ToolContext, city: str) -> p.ToolResult:
    # Your weather API logic here
    return p.ToolResult(f"Sunny, 72°F in {city}")

async def main():
    async with p.Server() as server:
        agent = await server.create_agent(...)
        await agent.create_guideline(...)
```

透過 `create_agent` 建立代理程式，並使用 `create_guideline` 定義行為準則 (包含條件、動作和使用的工具)。  框架會自動處理準則的匹配和執行，確保代理程式的行為符合預期。  最後，範例中展示瞭如何使用內建的 React 元件或自建前端來整合代理程式。  Parlant 提供完整的可解釋性，方便開發者瞭解代理程式的每個決策。
